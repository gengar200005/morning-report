새 ADR 작성. 주제: $ARGUMENTS

순서대로:

1. `docs/decisions/` 기존 파일 확인 → 다음 번호 정하기 (예: 001, 002 있으면 003)

2. 파일명: `NNN-[kebab-case-title].md`

3. 템플릿:

   ```markdown
   # ADR-NNN: [제목]

   **날짜**: YYYY-MM-DD
   **상태**: 채택 | 대체됨 | 폐기

   ## 배경
   [왜 이 결정이 필요했는지 — 기존 상태·문제점]

   ## 결정
   [무엇을 결정했는지 — 구체적이고 명확하게]

   ## 대안
   [검토한 다른 옵션들과 각각 택하지 않은 이유]

   ## 결과
   ### 긍정적
   [이 결정의 긍정적 영향]
   ### 부정적
   [트레이드오프·단점]
   ### 후속 작업
   [이 결정 후 필요한 추가 작업]
   ```

4. 작성 전 사용자 확인 받기. 맥락은 SESSION_LOG 또는 사용자 설명에서 가져오기.

5. 승인 후 파일 생성 + `CLAUDE.md`의 "최근 주요 결정" 섹션에 한 줄 추가.
