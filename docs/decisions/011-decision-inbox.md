# ADR-011: 결정 인박스 (Decision Inbox) 운영 원칙

**날짜**: 2026-04-28
**상태**: 채택
**관련**: 운영 규칙 (CLAUDE.md "브랜치 / 배포")

## 배경

Claude Code on the web 환경에서 **세션이 도중에 뻗어 컨텍스트 손실 발생** 4번
연속 관찰 (2026-04-22 ~ 04-28). 패턴:

1. 세션 중 마스터 + Claude 가 "해보자" 방향성 결정 (예: KOSPI 200 추적)
2. 코드/문서 변경 commit 전 세션이 죽음
3. 다음 세션이 git log 에서 결정 흔적 못 찾음 → 마스터 입에 의존
4. 매번 컨텍스트 회복에 5-10분 소요 + 누락 위험

특히 **결정만 하고 코드 작업 안 한 케이스** (예: 04-28 직전 KOSPI 200 결정 후
즉시 죽음) 는 git history 에 흔적 0 → session-start 스킬 의 brunch drift 체크도
못 잡음.

## 결정

`docs/decisions/_inbox.md` 한 파일을 두고, 다음 운영 원칙 채택:

### 1. 기록 시점

세션 중 **"해보자" 가 굳어지는 순간** (= 의논 후 기각 아니라 실행 의도가 명확
해진 순간) 1~3줄 즉시 추가.

### 2. 즉시 commit + push

코드 변경 0 이어도 OK. 본 파일에 한해 운영 규칙 "Push 는 마스터 명시 승인 시"
의 **예외 허용** (자동 push OK).

### 3. 다음 세션 회수

세션 시작 시 (`session-start` 스킬 또는 수동) `_inbox.md` 확인 → 해당 entry 들을
정식 문서 (`CLAUDE.md` "현재 상태" / `SESSION_LOG.md` / 신규 ADR) 로 흡수 →
흡수 완료된 entry 는 `_inbox.md` 에서 삭제.

### 4. 빈 상태 유지

`_inbox.md` 의 `---` 구분선 아래는 평소 빈 상태. 다음 세션이 빈 파일 보고 시작
하는 것이 정상 (= 이전 세션이 깨끗하게 마감).

## 근거

### 세션 뻗음 mitigation 후보 비교

| 후보 | 효과 | 비용 | 위험 |
|---|---|---|---|
| **결정 인박스 + 자동 push 예외** ⭐ | 높음 (직접 mitigation) | 낮음 (1 파일 추가) | 낮음 (자동 push 한 파일 한정) |
| session-start 스킬 보강 | 중간 (drift 못 잡는 케이스 있음) | 중간 (스킬 수정) | 낮음 |
| Stop hook 자동 commit | 높음 | 높음 (스크립트 + 안전장치) | 높음 (의도치 않은 commit) |
| 매번 사용자에게 알림 | 낮음 | 0 | 0 |

결정 인박스가 효과/비용 비율 가장 좋음. 자동 push 예외는 한 파일 한정이라
운영 규칙 정신과 충돌 작음.

### 자동 push 예외 정당성

운영 규칙 "Push 는 마스터 명시 승인 시" 의 의도 = 라이브 시스템에 의도치 않은
변경 밀어버림 방지. `_inbox.md` 는:

- 텍스트 메모만 있는 파일, 라이브 시스템 영향 0
- 매번 마스터 승인 받으면 결정 즉시 push 의 실효성 저하
- 한 파일 한정 예외라 운영 규칙 단순성 손상 작음

### 결정 vs 의논 구분

본 ADR 는 **"해보자" 가 굳어진 결정만** 기록 대상. 의논 단계 (가설 검토 / 대안
나열) 는 기록하지 않음. 4번 연속 기각 패턴 (ADR-005/004/001/010) 의 narrative
bias 위험을 _inbox 에 누적시키지 않기 위함.

## 결과

### 기대 효과

- 세션 뻗음 → 다음 세션 회복 시간 5-10분 → ~1분 (git log + _inbox 만 확인)
- 결정 추적 자동화 (수동 SESSION_LOG 갱신 의존 감소)
- 누락 위험 0

### 부작용

- **자동 push 예외 1건 신설** — 운영 규칙 단순성 약간 손상
- **메모 누적 위험** — 다음 세션이 회수 안 하면 stale entries 쌓임. 회수 의무는
  `session-start` 스킬에 명시 권장 (후속).
- **narrative bias 재발** 가능성 — "해보자" 결정 빈도 증가 시 ADR-010 메타 원칙
  과 충돌. 분기 점검 (3개월) 권장.

## 검토한 대안 (기각)

- **Stop hook 자동 commit + push**: 효과 ↑↑ 이지만 의도치 않은 commit 위험. 라이브
  코드 변경까지 자동 push 되면 사고 발생.
- **session-start 스킬에 "결정 회수" 단계 추가**: 본 ADR 와 보완 관계, 별도 ADR
  대상 아님. 후속 작업.
- **CLAUDE.md 안에 inbox 섹션 두기**: CLAUDE.md 가 자동 push 면 라이브 영향 가능
  성 + 파일 비대화. 별도 파일 분리가 깔끔.

## 미래 폐기 조건

다음 중 하나면 본 ADR 폐기:

1. **3개월 운영 후 _inbox 활용도 0** — 결정 인박스가 실제 회복에 도움 안 되면
   유지 가치 없음.
2. **자동 push 예외가 사고 유발** — 의도치 않은 _inbox commit 으로 main 영향
   발생 시 즉시 운영 규칙 강화 + 본 ADR 폐기.
3. **세션 안정성 자체 개선** — Claude Code on the web 가 세션 뻗음 빈도 ≤ 1회/월
   수준 안정화되면 mitigation 자체 불필요.
