# ADR-004: 섹터 게이트를 전략 진입 조건으로 통합 — **기각**

**날짜**: 2026-04-23
**상태**: 기각 (Rejected)
**관련**: ADR-002 (strategy 모듈), ADR-003 (섹터 강도 산정), `backtest/strategy.py`

## 배경

ADR-003 Amendment 2 (2026-04-23 #4) 에서 KOSPI200 11섹터 체계의 "주도+강세"
등급 섹터가 다음 1개월 universe-avg 대비 **+2.37%/월, hit 83%** 의 알파를
보이는 것이 회귀 검증됨. 자연스러운 후속 질문:

> 이 섹터 알파를 `kr_report.py` signals 생성 시 **진입 게이트**로 통합하면
> 전략 CAGR 이 개선되는가?

본 ADR 은 이를 11.3년 (2015-2026) × 162종목 백테로 실증 검증.

## 실험 설계

### 구현

1. `strategy_config.yaml::signal.sector_gate` 블록 신규
   - `enabled` / `tiers` / `fallback_on_na` / `recompute_every`
2. `strategy.py`:
   - `precompute_sector_tiers(all_dates, stock_arr, cfg)` — N거래일 단위로
     `sector_breadth.compute_sector_scores(end_date=D-1)` 재계산, 캐시
   - `check_sector_gate(ticker, i, cache, cfg)` — 후보 필터링 단계에 추가
   - 룩어헤드 방지: day i 판정은 `all_dates[i-1]` 종가까지만 사용
3. 마커캡은 2026-04 snapshot 고정 (섹터 내 상대 비중은 시간 안정적, 경미한
   look-ahead)

## 결정

**ADR-004 기각**. 모든 variant 가 baseline CAGR 를 하회.

### 이유

1. **전체 기간 CAGR 악화 일관됨** — 최소 손해 variant (D) 도 -2.99%p.
   가장 직관적 조합 (C, 주도+강세) 는 -8.12%p.
2. **MDD 악화** — C/D 모두 MDD 가 baseline -29.8% 대비 악화 (C: -33.3%,
   D: -34.1%). 필터로 거래수 줄면서도 리스크 감소 효과 없음.
3. **강세장 알파 손실 큼** — 2025+ 에서 C 는 -32.4%p, 심지어 D 도 -31.1%p.
   섹터 등급이 "주도/강세" 로 격상되는 시점엔 이미 상승 상당부분 진행된 상태.
4. **박스권에서만 개선** — 2015-19 에서 B/C 가 baseline 대비 +3.76~+11.54%p
   개선. D 는 효과 미미. 박스권 조건부 활성화가 가능할 수 있음 (후속 연구).

### 근본 가설 (왜 기각되는가)

**Minervini 8조건 자체가 trend-following 필터**이고 섹터 강도 역시
trend-following 이므로 두 필터는 높은 상관성을 가짐 → 추가 알파가 거의
없는 상태에서 **등급 부여 지연 (lag)** 만 비용으로 발생.

ADR-003 Amendment 2 의 "+2.37%/월 알파" 는 **섹터 단위 다음 달 평균 수익률**
기준이었고, **이미 Minervini 통과한 종목 부분집합** 에 대한 조건부 알파가
아니었음. 통합 시점에 이 조건 의존성이 드러남.

### E (strict) = C (pass) 이유

N/A 섹터에 속한 종목이 실제로 거의 없어 fallback 정책 차이가 결과를
바꾸지 않았음 (warmup 252일 이후 대부분 섹터가 등급 부여됨).

## 검토한 대안 (후속 연구 후보)

- **박스권 detection + 조건부 활성화** — 시장 regime (예: 6M KOSPI
  yearly return, MA200 slope) 기반으로 박스권 구간에만 게이트 활성화.
  2015-19 이득 보존 + 중립/강세장 손실 회피. 구현 1-2h, 검증 추가 백테.
- **게이트 대신 랭킹 가점** — 현재 RS 백분위 순위로 top N 선택하는 단계에서
  섹터 등급을 보너스 점수로 반영 (RS + 섹터 점수 가중합). 필터가 아닌
  ranking 알파로 사용. 구현 1h.
- **약세 섹터 차단만** — D 는 -2.99%p 악화지만 2015-19 조차 개선 실패.
  기각.
- **섹터 scoring 주기 단축** — recompute_every 5 → 1 로 낮추면 lag 감소
  가능. 단, 본 실험에서 "주도→강세→중립→약세" 플립은 대체로 주 단위라
  효과 미미할 듯. 검증 비용 낮으나 우선순위 낮음.

## 구현 상태

- `strategy_config.yaml` 에 `sector_gate` 블록 **존치** (enabled: false 기본값).
  다음 실험 (박스권 조건부, 랭킹 가점) 에서 인프라로 재사용.
- `strategy.py::precompute_sector_tiers` / `check_sector_gate` **존치**.
  미사용이면 no-op (enabled=false 시 즉시 return).
- `backtest/99_sector_gate_ab.py` / `99_sector_gate_variants.py` **존치** —
  대안 탐색 시 기반 템플릿.

## 영향

- **kr_report.py 통합 없음**. 라이브 스크리닝에 섹터 게이트 적용하지 않음.
- T10/CD60 확정 전략 **변경 없음** (+29.55% CAGR, MDD -29.83%, 거래 333건
  유지).
- 섹터 게이트는 "표시용 정보" 범위 유지 (ADR-003 범위). 리포트에서 주도
  섹터는 계속 강조하지만 거래 결정에는 반영 않음.

## 교훈

1. **회귀 알파 ≠ 전략 알파** — ADR-003 Amendment 2 의 섹터 단위 예측력은
   진짜였지만, Minervini 통과 종목에 대한 **조건부 알파** 와는 다른 차원.
   "A 조건이 B 수익과 상관 있음" 과 "B 전략에 A 필터 추가하면 개선" 은
   별개 명제.
2. **상관된 필터 중첩은 lag만 추가** — 같은 방향의 trend-following 지표
   두 개를 AND 로 걸면 신호 늦어질 뿐 알파 배가 안 됨. 서로 다른 방향
   (예: momentum × value, trend × mean-reversion) 의 필터 조합이 효율적.
3. **박스권 vs 중립장의 필터 효과 비대칭** — 박스권에선 false signal 많아
   필터 효과 큼. 상승장에선 필터가 true signal 까지 걸러내 알파 손실.
   regime-dependent gate 가 이론적으로 타당.
4. **확장 실험 비용 저렴** — tier_cache 1회 (80s) 계산 후 4개 variant
   1.5-2.6s 씩. 의사결정 전 변형 스윕이 거의 공짜.
