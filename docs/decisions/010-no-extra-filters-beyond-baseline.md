# ADR-010: baseline 외 추가 필터 추구 자제 원칙

**날짜**: 2026-04-26 (#2 세션)
**상태**: 채택 (메타 원칙)
**관련**: ADR-001 (T10/CD60 baseline), ADR-004 (섹터 게이트 기각),
ADR-005 (fresh-signal 기각), ADR-006 (streak≤10 기각)

## 배경

Phase 3 baseline (T10/CD60: Minervini 8조건 + 수급 프록시 + RS + 시장 게이트)
확정 후 2026-02 ~ 2026-04 에 걸쳐 추가 필터 가설 4건 검증. **모두 기각**:

| 가설 | 검증 시점 | 결과 |
|---|---|---|
| ADR-001 / VCP auto 3요소 | 103종목 시점 + 162종목 재검증 (04-26 #2) | 162: CAGR +29.55% → +8.20% (**-21.35%p**), 거래 -49% |
| ADR-004 / 섹터 게이트 무조건 | 04-23 5 variant 그리드 | 5 variant 전부 baseline 하회 |
| ADR-005 / fresh-signal entry timing | 04-24 walkforward | IS rank 상위 / OOS 방향 불일치 |
| 박스권 조건부 섹터 게이트 | 04-26 #2 6 variant 그리드 | 박스권 +10%p 보호 / 강세장 -8.93%p / MDD 악화 / robustness FAIL |

박스권 조건부 게이트는 별도 ADR 승격 없이 본 ADR-010 안에 검증 결과 흡수. 4번
연속 동일 패턴 fail 의 학습은 개별 ADR 보다 메타 원칙으로 보존하는 것이 효율적.

## 결정

baseline 외 추가 필터는 다음 **두 조건을 모두 만족할 때만** 백테 검증 시도:

1. **사전 가설 명시**: 알파 메커니즘이 백테 외 1차 출처에서 검증됨 (학술 논문 /
   O'Neil / Minervini / Weinstein / Asness 등). "이론적으로 그럴듯하다" 또는
   "직관적으로 맞을 것 같다" 만으로는 부족.
2. **Robustness 사전 plan**: 임의 임계값 ±50% 범위에서 sensitivity 그리드 사전
   설계 + 모든 점에서 baseline -1%p 이내 통과 조건 명시.

두 조건 모두 통과 못하면 **백테 검증 자체 자제**. 추가 필터 추구는 시간 비용 +
narrative 끌림 + overfit 위험을 발생시키며, 4번 연속 fail 한 패턴은 다음 시도도
같은 결과 가능성이 높음.

## 근거

### 4 case 공통 fail 메커니즘

baseline 알파 (CAGR +29.55%, 11.3년, 162종목) 는 다음 4가지의 곱:

```
RS 모멘텀  ×  Minervini 8조건  ×  수급 프록시  ×  시장 게이트
```

추가 필터를 얹으면:

| 효과 | 측정 |
|---|---|
| 진입 정밀도 ↑ | 승률 / 손익비 일부 개선 (+0.1~0.3x) |
| **진입 횟수 ↓** | **-30~49%** (filter 별 강도 따라) |
| 알파 큰 추세 진입 차단 | 강세장 (2025+) 알파 -8~32%p 손실 |

→ **손익비 개선 < 진입 차단의 알파 손실**. 순 algorithm 깎음.

### Narrative 끌림 위험

- "Minervini SEPA 의 VCP 트리거" "Weinstein Stage 보조" "박스권 보호" 등 모두
  **이론적으로 그럴듯한 narrative**. 실제로 검증 통과한 baseline 룰들도 동일한
  출처에서 나옴.
- 차이는 **baseline 룰들은 backtest 통과**, 추가 필터들은 fail.
- ADR-005 의 핵심 진단 ("추가 필터 baseline 하회") 이 4 case 공통.
- ADR-009 Claude augmentation 폐기도 같은 결: narrative 가치 < narrative 끌림 위험.

### Robustness sensitivity 가 데이터 마이닝 차단

박스권 조건부 게이트 (04-26 #2) 의 경우:
- ±10% return / 0.05% slope 한 점에서만 baseline -0.45%p 동등
- ±5% (-2.33%p) / ±15% (-1.28%p) 두 sensitivity 점은 baseline -1%p 초과
- → **단일 임계값 통과는 우연**. 5개 점 (예: ±5/10/15% × slope 0.02/0.05) 모두
  통과해야 robust 판정.

이 robustness 게이트가 4 case 모두에 적용됐다면 사전에 모두 거를 수 있었음.
미래 시도에는 사전 plan 단계에서 robustness 그리드 명시 의무화.

## 대안

### ❌ 추가 필터 검증 자유롭게 시도

- 검증 자체에 시간 (5-30분/회) + narrative 잔류 + 마스터 의사결정 비용
- 4 case fail 후 사후적으로 "그럴 줄 알았다" 가 되는데 검증 자체가 narrative 끌림
  근원 (= 박스권 게이트 +10%p 가 보호 효과 진짜라는 확신)
- 미래 검증 안 하면 모를 수 있는 정당한 알파 후보 차단 위험은 사전 게이트 두 조건이
  필터링

### ❌ 모든 추가 필터 영구 차단

- 미래 진정한 알파 후보 (예: 새 팩터 학술 발견, 시장 구조 변화) 도 차단
- 너무 보수적, 학습 자체 멈춤

### ❌ 박스권 게이트 채택 (검증 +10%p 만 보면 매력)

- 박스권 보호 +10%p 는 실재
- 그러나 강세장 false positive -8.93%p + MDD 악화 + robustness FAIL → 종합 알파
  음수
- 단일 임계값 통과를 일반화하면 ADR-005 패턴 (overfitting / data mining) 의
  변종

### ✅ 두 조건 사전 게이트 (채택)

- 사전 검증 + robustness 이중 게이트로 fail 패턴 직접 차단
- 진정한 알파 후보 (사전 검증 + robust) 는 통과 가능
- 4 case 의 공통 fail 메커니즘 (narrative 끌림 + 단일 임계값) 두 곳 모두에서 거름

## 결과

### 긍정적

- 미래 5번째 추가 필터 유혹 시 사전 게이트 두 조건으로 시간/narrative 비용 절감
- 페이퍼 트레이딩 / 실전 운영 / 인프라 안정화에 자원 집중
- 박스권 보호 +10%p 같은 부분 효과를 게이트 외 다른 채널 (sizing / risk
  parameter) 로 구현하는 방향이 정당화 (게이트 통합과 다른 메커니즘 = 다른 가설)

### 부정적

- "사전 검증" / "robustness plan" 판단이 주관적. 조건 충족 여부에 마스터 + Claude
  의 합의 필요.
- 진정한 알파 후보가 학술 발견 직후 사전 검증 부족 단계에서 일시 차단 가능. 단,
  baseline +29.55% 가 충분히 높아 기다림 비용 작음.

### 박스권 보호 효과 (+10%p in 2015-19) 의 별도 활용 여지

본 검증의 부산물:
- regime detection 자체는 작동 (lookahead bias 없음, 분포 47.8% sideways /
  32.2% bull / 20.0% bear)
- 박스권 시 게이트 외 채널로 보호 가능성: (a) stop_loss 완화 (-7% → -5%) (b)
  진입 슬롯 축소 (5 → 3) (c) sizing 차등 (RS 점수 가중)
- 이는 "추가 필터" 가 아니라 **risk management 파라미터 조정** 영역 → 본 ADR
  적용 대상 아님. 페이퍼 트레이딩 후 검토 가능.
